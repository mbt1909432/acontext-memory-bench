# Acontext 长期记忆能力测试方案

> 基于 LongMemEval (ICLR 2025) 基准测试，评估 Acontext 在 AI Agent 长期记忆管理方面的能力

**文档日期：** 2026年3月12日
**状态：** 准备就绪

---

## 📋 项目背景

### 目标

使用学术界公认的长期记忆基准测试 LongMemEval，客观评估 Acontext 在 AI Agent 记忆管理方面的能力，为产品优化和市场推广提供数据支撑。

### 什么是 LongMemEval？

LongMemEval 是 ICLR 2025 发表的基准测试，专门用于评估聊天助手的**长期记忆能力**。它模拟真实场景中用户与 AI 的多轮对话历史，测试 AI 能否准确回忆并利用历史信息回答问题。

| 指标 | 数值 |
|------|------|
| 测试问题 | 500 |
| 核心能力维度 | 5 |
| 历史会话/问题 | 40+ |

---

## 🧠 测试的核心能力

| 能力维度 | 描述 | 示例场景 |
|----------|------|----------|
| **信息提取** | 从历史对话中提取特定信息 | "我之前说的毕业院校是哪所？" |
| **多会话推理** | 跨多个对话进行信息整合 | "对比我第一次和第三次提到的偏好" |
| **知识更新** | 处理用户偏好的变化 | "我现在喜欢什么？"（之前改过） |
| **时间推理** | 基于时间戳进行推理 | "上次修改密码是几天前？" |
| **弃权判断** | 识别无法回答的问题 | "我的猫叫什么？"（从未提过） |

## 🔬 测试方案

### 测试流程

```
加载测试数据 → 存入 Acontext → 学习/检索 → 回答问题 → 评估结果
```

### 对比实验设计

> **📌 说明：** LongMemEval 论文已有 GPT-4o 基线结果（准确率 80%+），无需重新跑基线。我们的测试重点是验证 **Acontext 的记忆方案**效果。

| 方案 | 做法 | 结果来源 |
|------|------|----------|
| 基线 | 直接将全部历史对话喂给 LLM | ✅ 论文已有（GPT-4o ~80%+） |
| **Acontext 测试** | Session Storage 存对话 → Learning Space 提取记忆 → 检索回答 | 🧪 我们要测的 |

### 核心验证点

1. **记忆提取能力**：Learning Space 能否从对话中自动提取有效记忆（用户偏好、事实）
2. **检索效果**：Acontext 检索到的相关记忆 vs 全量历史，准确率差距多少
3. **Token 效率**：Acontext 方案需要多少 token vs 全量历史（成本对比）

### 评估指标

- **准确率 (Accuracy)**：与论文基线对比
- **Token 消耗**：Acontext 方案 vs 全量历史的成本对比
- **记忆召回率**：检索到的相关会话占比

## 🏗️ 技术架构

```
LongMemEval 数据集
       ↓
┌──────────────────────────────┐
│  Step 1: Session Storage     │  ← 存储用户-助手历史对话
└──────────────────────────────┘
       ↓
┌──────────────────────────────┐
│  Step 2: Learning Space      │  ← 从对话中自动提取用户偏好、事实、技能
└──────────────────────────────┘
       ↓
┌──────────────────────────────┐
│  Step 3: LLM (GPT-4o)        │  ← 基于记忆回答问题
└──────────────────────────────┘
       ↓
┌──────────────────────────────┐
│  评估系统                    │  ← GPT-4o 作为裁判，判断答案正确性
└──────────────────────────────┘
```
